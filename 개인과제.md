## 캐시
  - GPU 캐쉬
  - Disk 캐쉬
  - web 캐쉬
  - 검색 엔진 캐쉬
  - 데이터베이스 캐쉬
  - **CPU 캐시**
    - 기능
      * CPU가 메인메모리에 접근하는 시간을 줄이기 위해 사용
      * 자주 사용하는 MM내의 데이터를 저장
      
    - 레벨
      * 레벨이 낮을수록 CPU와 가깝지만 용량은 줄어듬
      * L1 Cache : CPU에 내장되어 데이터 사용 및 참조에 이용(SRAM 사용)
      * L2 Cache : L1 캐시에서 찾지 못하였을 때 넘어가는 다음 캐시(DRAM 사용)
    
    - SRAM
      * 전원 공급이 없는 한 내용 유지
      * 입출력에 걸리는 시간 일정
      * 장점
        > A. 전력 소모가 적음  
        > B. 초기화 필요 없음  
        
      * 단점
        > A. 용량이 작음  
        > B. 단가가 비쌈  
        > C. 전력 소모량이 일정하지 않음  
        
    - DRAM
      * 전력을 계속해서 주지 않으면 기억이 사라짐
      * 메모리 초기화가 지속적으로 필요
      * 장점
        > A. 단가가 쌈  
        > B. 용량이 큼  
        
      * 단점
        > A. 전력소모가 심함  
        > B. 초기화를 지속적으로 해줘야함  
    
## 데이터베이스
  - In-memory 데이터베이스 : 메인메모리에 있어 응답 속도가 빠르다
  - Active 데이터베이스 : 데이터베이스 내,외부의 변화에 반응하여 실행(알람 등)
  - Deductive 데이터베이스 : 논리과정을 통해 다른 데이터베이스와 결합
  - Distributed 데이터베이스 : 데이터와 DBMS가 여러 컴퓨터에 분포된 데이터베이스
  - Document-Oriented 데이터베이스 : 문서 파일 관리에 이용
  - Embedded 데이터베이스 : 소프트웨어 앱이 보관되어 있는 데이터베이스
  - Temporal 데이터베이스 : 시간과 그 시간에 관련된 사건들을 정리한 데이터베이스
  - Unstructured 데이터베이스 : 이전의 정의된 방식대로 정리되지 않은 데이터베이스
  - Spatial 데이터베이스 : 기하학적 영역에서 데이터 보관과 조회
  - Real-Time 데이터베이스 : 현재 진행되고 있는 작업을 보관
  - Parallel 데이터베이스 : 병렬 컴퓨터로 다양한 동작을 수행
  - Operational 데이터베이스 : 현재 진행되고 있는 직업을 수정
  - Mobile 데이터베이스 : 모바일 디바이스에 저장
  - Hypertext 데이터베이스 : 하이퍼링킹에 관해 정보를 저장하고 있는 데이터베이스
  - Graph 데이터베이스 : 그래프 구조를 사용하여 데이터를 분석 및 저장
  - Federated 데이터베이스 : 여러 데이터베이스를 하나로 묶는 데이터베이스
  - **Cloud 데이터베이스**
    - Virtual-Machine Database
      * 사용자가 직접 VM을 만들어 데이터베이스를 관리
        > VM
          * 가상의 구조물이 실제 물리적인 기계와 같은 역할을 하는것
          * 하드웨어, 소프트웨어 등이 VM이 될 수 있음
          
    - Database as a Service
      * 어플리캐이션 소유주가 직접 데이터베이스를 관리하는 형식
      * Naver의 N-Drive, 구글의 Drive, Dropbox 등이 이에 해당
      
    - 특징
      * 웹 기반으로 운영된다
      * API를 기반으로 데이터베이스의 자료들을 관리한다
        > A. API : 데이터베이스 관리 시스템과 운영 프로그램이 통신 할 때 사용하는 언어
        
      * 데이터베이스는 다른 소프트웨어, OS 등으로 만들어짐
        > A. 이에 대한 지속적인 관리가 필요(업데이트, 패치 등)
    
    - 데이터베이스 언어
      * SQL(Structed Query Language)
        > A. 특정한 데이터에 한정되어 쓰이지 않음  
        > B. 관계형 데이터베이스 관리 시스템  
        > C. 1973년 SQUARE에서 출발, 오라클의 SQL로 발전  
        > D. 데이터 정의와 조작 가능  
        > E. 엑세스 경로를 참고하지 않음  
        > F. 테이블 단위로 연산 수행  
        
      * NoSQL(non SQL)
        > A. 비관계형 데이터베이스 관리 시스템  
        > B. 대규모의 데이터를 유용하게 처리 가능  
        > C. 단순 검색 및 추가 작업을 위해 용이  
        > D. 지연과 처리율 우수  
        > E. 수평적 확장 가능
